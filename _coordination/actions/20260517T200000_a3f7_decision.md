# F1 Minimal Phase Launch Decision

## Summary

User 授權啟動 F1 minimal phase；目標是定義 action file schema 並開始 selective reading 機制。

## Context

依 Codex 32B audit 順序建議（維護債 → F1 → F2 → Platform Hook → F3 → F4），維護債已落實。

User 於本對話以「進下一步」明文授權 F1 啟動，並逐項回應 F1-A 到 F1-H 8 個 scope 決策：

- F1-A：路徑命名規則採 `<timestamp>_<action_id>_<type>.md`
- F1-B：8 種 type 閉集
- F1-C：依 Codex P2.4 拆分 record_status + decision_effect
- F1-D：6 Codex 欄位 + phase_ref
- F1-E：寫 1 個 sample（即本檔）
- F1-F：Schema 文件落 workflow_notes/
- F1-G：PowerShell 草案 index regen helper
- F1-H：走完整循環（後續 Codex audit）

## Action Taken

- 設計 F1 minimal schema（21 個 frontmatter 欄位、8 type、2 status 維度）
- 寫本檔作為 schema dog-fooding 範例
- 寫 `_coordination/actions/index.md` 初始版本（僅本檔 1 列）
- 寫 `_coordination/workflow_notes/20260517_32_c_f1_minimal_schema_design.md` 完整設計

## Result / Outcome

F1 minimal phase 啟動完成；schema 設計就緒；首個 action file 落地。

後續：user 審 → standard gate 落檔 → user-side commit + push → 送 Codex audit。

## Source Limitations

- 本檔對應 schema design v1.0-candidate；未來 schema 演進靠 supersede 鏈
- index 為手寫初始版，待 PowerShell helper 跑過才 auto-maintained
- F1 minimal 不含 validator，schema violations 不會被自動擋
- `created_at` 為近似值（對話內無精確時鐘）
