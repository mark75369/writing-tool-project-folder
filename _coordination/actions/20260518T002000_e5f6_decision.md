# Decision: F2 Minimal Schema Pushback Finalization

## Summary

User 對 Codex 32E 校核 response 作 4 條 pushback，全採 active controller 提供的子選項建議。F2-A 到 F2-H 子決策已 finalize（屬 pre-launch design decisions finalized as REVIEW evidence）；F2 strict gate 啟動仍待 user 明文授權。

## Context

Codex 32E response 提出 3 refinement + 3 design contribution。Active controller 列出 4 個 user 可能想 push back 的點（中性陳述）並提供各自子選項建議。User 回「pushback 都依照你的建議」，4 條 pushback finalize。

## Action Taken / Decisions

### Pushback #1 — handoff_type

- User 選擇:4 種（拒 Codex R1 的 3 種收斂）
- 子選項:4-C hybrid（4 enum + `platform_from/to` + `subject_from/to` 拆分）

### Pushback #2 — T6 範圍

- User 選擇:採 active controller 重寫
- 子選項:T6 拆 T6a + T6b
  - T6a:引用 F1 schema §12/§13 延後 / Not Approved 列表
  - T6b:試圖建立 `handoffs/` 獨立 folder（違反整合-1）

### Pushback #3 — handoff_status

- User 選擇:簡化
- 子選項:S-B（4 種，砍 `expired`，由 `expires_at` 推導）
- 4 種:proposed / accepted_for_startup / rejected / superseded

### Pushback #4 — current_state_sync

- User 選擇:必填 vs optional → 採 X 必填
- 子選項:X（Codex 原版必填三值 update_required / no_change / blocked）

## Result / Outcome

F2-A 到 F2-H 子決策 finalize（見 32E §3 整合 + 本 action 上方）。Strict gate F2 minimal launch 仍待 user 明文授權；本 decision 為其 evidence input。

## Source Limitations

- 本 decision 為 user 在 chat 中的選擇，未經 strict gate
- Active controller 子選項建議為 advisory，不是 authority
- F2 strict gate 啟動時 user 仍可 revise 本 decision

## Stop Conditions

- 本 decision 被當 F2 strict gate 已通過 → stop
- 本 decision 被當 schema LOCKED → stop
- 本 decision 內任一 pushback 被擴張到 32E 以外的 scope → stop

## Boundary

本 decision 不批准:

- F2 minimal strict gate 啟動
- F2 schema 完整版設計檔（32F）建立（仍需 strict gate）
- HARNESS Core / 32A-LOCK 修改
- Mode B 切換
- commit / push / remote 設定
- protected content read
- Runtime / SDK 啟動
- room activation

本 decision 僅代表「user 接受 4 個 pushback 子決策作為 F2 launch evidence」，不是 F2 strict gate 通過。
